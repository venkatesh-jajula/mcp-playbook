# Ways to Build MCP Servers

There are two main ways to build an MCP server in Python:

## 1. Low-Level — Official SDK (`Server` class)

The official approach from Anthropic. You manually define handlers and wire everything together yourself. Gives you full control over the protocol, but requires a lot of boilerplate code — complex and verbose even for simple servers.

## 2. High-Level — FastMCP

A Pythonic wrapper built on top of the official SDK, inspired by FastAPI. You just decorate your functions and FastMCP handles the rest — schemas, handlers, everything. Dramatically less code for the same result.

> FastMCP started as a community project but grew so fast (nearly matching the official SDK in GitHub stars within 6 months) that Anthropic absorbed it into the official SDK. It now ships built-in — no separate install needed.

## Which to use?

Use **FastMCP** for almost everything. Use the **low-level SDK** only if you need deep protocol control.
